12  * This header is the interface of platform storage API.
13  * The user will be able to write, read and delete files in platform storage via the provided interfaces.
14  * The user shall use this interface to interact with the following modules:
15  *
16  * 1) Secure Storage
17  * 2) Customer Partition
18  * 3) Linux File System
19  *
20  * The user does not need to add any other header to use this API.
21  *
22  * The following functionalities are provided by the API.
23  *
24  * # Description of Configure function
25  * This function allows the user to configure the platform storage API at startup
26  *
27  * # Description of Write function
28  * This function allows the user to write a single file to platform storage.
29  * The user must specify the platform storage type. (Where should we write this file to?)
30  * The user must specify the name of the file. (What is the name of the file?)
31  * The user must specify the content of the file. (What is the content of the file?)
32  *
33  * # Description of Read function
34  * This function allows the user to read a single file from platform storage.
35  * The user must specify the platform storage type. (Where should we read this file from?)
36  * The user must specify the name of the file. (What is the name of the file?)
37  * The user must specify a string to copy the content to. (Where should we copy the content to?)
38  *
39  * # Description of Delete function
40  * This function allows the user to delete a single file from platform storage.
41  * The user must specify the platform storage type. (Where should we delete this file from?)
42  * The user must specify the name of the file. (What is the name of the file?)
43  *
44  * The following use cases are supported by the API.
45  *
46  * # Storing a single file with plain text
47  * The user stores a single file with plain text in the root directory of a type of platform storage.
48  *
49  * # Storing a single file with encoded text
50  * The user stores a single file with encoded text in the root directory of a type of platform storage.
51  *
52  * # Storing a single file with encrytped text
53  * The user stores a single file with encrypted text in the root directory of a type of platform storage.
54  *
55  */
