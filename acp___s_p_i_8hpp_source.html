12  * This header is the interface of SPI API.
13  * The user will be able to write to and read from slave devices over SPI via the provided interfaces.
14  * SPI stands for Serial Peripheral Interface. It is a synchronous serial communication interface
15  * specification used for short-distance communication. SPI devices can communicate in half duplex (HD)
16  * or full duplex (FD) mode using a master-slave architecture with a single master.
17  *
18  * 1.1 Half-duplex mode (HD)
19  * Half-duplex mode means the communication is one-sided. In this case it indicates that the master
20  * is not also reading while it is writing to the slave.
21  *
22  * 1.2 Full-duplex mode (FD)
23  * Full-duplex mode means that both the master and slave are reading and writing simultaneously.
24  *
25  * The user does not need to add any other header to use this API.
26  *
27  * The following functionalities are inherited from SerialBus class.
28  *
29  * # Description of GetFileDescriptor function
30  * This function allows the user to get the reference to an open device
31  *
32  * # Description of IsValid function
33  * This function allows the user to check if the file descriptor is valid
34  *
35  * # Description of GetFileName function
36  * This function allows the user to get the name of the resource to open (i.e. /dev/ttyAT)
37  *
38  * The following functionalities are provided by the API.
39  *
40  * # Description of Configure function
41  * This function allows the user to change the settings on the SPI bus
42  * The user must specify the SPI mode to be used.
43  * The user must specify the number of bits per byte.
44  * The user must specify the transfer speed in Hertz.
45  * The user must specify the transmission endianness*.
46  *
47  * Endianness indicates the ordering of bytes within a multi-byte number. For example, consider the
48  * unsigned hexadecimal number 0x1234, which requires at least two bytes to represent. In a little-endian
49  * ordering, the bytes would be arranged [0x34, 0x12], while in a big-endian ordering they would be
50  * [0x12, 0x34].
51  *
52  * # Description of WriteSerial function (1)
53  * This function allows the user to write a string to the SPI bus in half-duplex mode
54  * The user must specify the string to be written.
55  *
56  * # Description of WriteSerial function (2)
57  * This function allows the user to write a C array to the SPI bus in half-duplex mode
58  * The user must specify the data array to be written.
59  * The user must specify the number of bytes to be written.
60  *
61  * # Description of WriteRegister function
62  * This function allows the user to write to a register over the SPI bus in half-duplex mode
63  * The user must specify the address of a register within the slave device.
64  * The user must specify the single-byte value to be written.
65  *
66  * # Description of ReadSerial function
67  * This function allows the user to read from the SPI bus in half-duplex mode
68  * The user must specify the array where input will be copied.
69  * The user must specify the number of bytes to be copied.
70  *
71  * # Description of ReadRegister function
72  * This function allows the user to read from a register over the SPI bus in half-duplex mode
73  * The user must specify the address of a register within the slave device.
74  * The user must specify the single-byte variable where input will be copied.
75  *
76  * # Description of WriteReadHD function
77  * This function allows the user to read from and to write to the SPI bus in half-duplex (HD) mode
78  * Half-duplex mode indicates that the master is writing to and reading from a slave in in two separate
79  * transactions, not simultaneously.
80  * The user must specify the data array to be written.
81  * The user must specify the number of bytes to be written.
82  * The user must specify the array where input will be copied.
83  * The user must specify the number of bytes to be copied.
84  *
85  * # Description of WriteReadFD function
86  * This function allows the user to read from and to write to the SPI bus in full-duplex (FD) mode
87  * Full-duplex mode indicates that both the master and slave are reading and writing simultaneously.
88  * The user must specify the data array to be written.
89  * The user must specify the array where input will be copied.
90  * The user must specify the size in bytes of both arrays.
91  * This function requires that both arrays are the same length
92  *
93  * The following use cases are supported by the API.
94  *
95  * # Configure SPI bus settings
96  * The user changes the settings on the SPI bus
97  *
98  * # Reading information from a slave device (chip) over SPI
99  * The user reads information from a slave device (chip) over SPI.
100  *
101  * # Writing a command to a driver over SPI
102  * The user writes a command to a driver over SPI.
103  *
104  * # Simultaneous read and write operation over SPI
105  * The user reads from and writes to the SPI bus in full-duplex (FD) mode (i.e. simultaneously)
106  */
