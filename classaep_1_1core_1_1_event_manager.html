Detailed Description

This is the core code for publish-subscribe event management model. Each message/update/action is a kind of event. Each communication channel can be abstracted as a event queue. Communication between modules can be simplified as publishing events to a queue so that other modules that are interested can subscribe. This model reduces the complexity of multiple-to-one, one-to-multiple and multiple-to- multiple communication.

To use the code

  1. register a queue by AddQueue a. To register one of the default queue from EventQueueID, just use the queue ID b. To register a custom queue that is not in EventQueueID, use a large enough queue ID to avoid conflict
  2. subscribe to a queue a. To fully leverage the power of this model, derive a class from EventConsumer b. You have flexibility to define what is interesting and what is not in EventShouldBeConsumed function c. Subscribe the queue using the derived consumer d. For individual event subscription, subscribe using AnyEventConsumer e. For lazy subscription to an individual event, call subscribe with event ID and callback handler
  3. at other places, publish to the queue a. If the queue is known, publish the event by the queue ID b. If the queue is unknown, publish the event by the event ID. Default queue will be retrieved.
