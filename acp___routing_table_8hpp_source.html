12  * This header is the interface of platform routing table API.
13  * The user will be able to perform basic routing functionalities via the provided interfaces.
14  *
15  * The user does not need to add any other header to use this API.
16  *
17  * The following functionalities are provided by the API.
18  *
19  * # Description of Add route function
20  * This function allows the user to add a IPv4 or IPv6 static route to a destination host on a given table.
21  * The user must specify the IP of the destination host.
22  * The user must specify the gateway of the route.
23  * The user must specify the device interface to which packets for this route will be sent.
24  * The user must specify the preferred IP protocol
25  * The user can provide empty table name if operation is expected on the main routing table
26  *
27  * # Description of Add default route function
28  * This function allows the user to add a IPv4 or IPv6 default route on a given table.
29  * The user must specify the gateway of the route.
30  * The user must specify the device interface to which packets for this route will be sent.
31  * The user must specify the preferred IP protocol
32  * The user can provide empty table name if operation is expected on the main routing table
33  *
34  * # Description of Delete route function
35  * This function allows the user to delete a IPv4 or IPv6 static route to a destination host from a given table.
36  * The user must specify the IP of the destination host.
37  * The user must specify the gateway of the route.
38  * The user must specify the device interface to which packets for this route were sent.
39  * The user must specify the preferred IP protocol
40  * The user can provide empty table name if operation is expected on the main routing table
41  *
42  * # Description of Delete default route function
43  * This function allows the user to delete a IPv4 or IPv6 default route from a given table.
44  * The user must specify the gateway of the route.
45  * The user must specify the device interface to which packets for this route were sent.
46  * The user must specify the preferred IP protocol
47  * The user can provide empty table name if operation is expected on the main routing table
48  *
49  * # Description of Create custom policy routing table function
50  * This function allows the user to create a custom routing table on the system.
51  * The user must specify the name of the routing table.
52  * The user must specify the id of the table
53  *
54  * # Description of Flush routing table function
55  * This function allows the user to flush/empty the routes from a routing table including routing cache.
56  * The user must specify the name of the routing table if not cache.
57  *
58  * # Description of Read routing table function
59  * This function allows the user to read the current routes from a routing table.
60  * The user must specify the name of the routing table.
61  * The user must specify the string where the routes would be copied to
62  *
63  * # Description of Show/Read Routing Policy DataBase (RPDB) rule function
64  * This function allows the user to read all the routing rules from the current RPDB.
65  * The user must specify the string where the rules would be copied to
66  *
67  * # Description of Add Or Delete IP rule on RPDB
68  * This function allows the user to add or delete custom routing rules on RPDB.
69  * The user must specify the list of selector options with associated values. Option NOT does not require any value.
70  * The user must specify the list of action options with associated values.
71  *
72  * # Description of Flush on RPDB
73  * This function allows the user to flush all the rules from the RPDB inlcuding main and default tables.
74  *
75  * # Description of Save on RPDB
76  * This function allows the user to save the current RPDB rules to a file (human unreadable raw data) to be restored later.
77  *
78  * # Description of Restore on RPDB
79  * This function allows the user to restore rules from a file.
80

82  * The following use cases are supported by the API.
83  *
84  * # Create a routing table
85  * The user creates a custom routing table on the system.
86  *
87  * # Read a routing table
88  * The user reads the routes from a routing table present on the system.
89  *
90  * # Add IPv4 and IPv6 routes
91  * The user adds default or static routes to a routing table present on the system.
92  *
93  * # Delete IPv4 and IPv6 routes
94  * The user deletes default or static routes from a routing table present on the system.
95  *
96  * # Flush the kernel routing cache or a routing table
97  * The user must provide the name of the routing table
98  * The user flushes the kernel routing cache or the existing routes from a routing table.
99  * Flushing the routing cache should be done after any routing table modifications
100  *
101  * # Show current RPDB
102  * The user views the current kernel RPDB contents.
103  *
104  * # Add or Delete custom routing rule to the RPDB
105  * The user adds or deletes rules to/from the RPDB to route packets using custom rules.
106  *
107  * # Flush all routing rule from the RPDB
108  * The user flushes all the rules from the RPDB.
109  * Causion should be maintained since it flushes all the rules from main and default routing tables as well.
110  * Adding rules for main and default routing tables is a good and safe practice after flushing to avoid any type of disconnection.
111  *
112  * # Save and Restore the RPDB
113  * The user saves the current RPDB rules to restore them later.
114  */
