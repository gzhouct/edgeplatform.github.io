13  * This header is the interface of the core logging API.
14  * The user will be able to perform the following logging operations:
15  * - set a log level,
16  * - set the name of the logfile to write to,
17  * - turn on logging to the console (in addition to the log file),
18  * - turn off logging to the console,
19  * - log a message to logfile containing the function name and a tag,
20  * - log a message to string buffer containing the function name and a tag,
21  * via the provided interfaces.
22  * The user shall use this interface to interact with the following modules:
23  * - All ACP classes can use this to log their data
24  *
25  * The user does not need to add any other headers to use this API.
26  * However each app (HUB, DAQ, OTA, etc.) may define their own header to log their own messages.
27  *
28  * The following functionalities are provided by the API:
29  *
30  *
31  * # Description of SetLogLevel function:
32  * This function allows the user to set the minimum log level to log.
33  *
34  * # Description of SetLogFile function:
35  * This function allows the user to set the name of the logfile to write.
36  *
37  * # Description of SplitToConsoleOn function:
38  * This function allows the user to send log messages to the console as well as the log file.
39  *
40  * # Description of SplitToConsoleOff function:
41  * This function allows the user to send log messages to only the log file.
42  *
43  * # Description of LogMessage function:
44  * This function allows the user to send a message to the logfile with optional function name and a tag
45  *
46  * # Description of LogBuferedMessage function:
47  * This function allows the user to send a message to a string buffer with optional function name and a tag
48  *
49  */
