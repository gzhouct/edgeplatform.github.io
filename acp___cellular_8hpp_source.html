12 * This header is the interface of platform cellular API.

14 *
15 * The user will be able to perform cellular functionalities via provided interfaces.
16 * If user wants to manage routing table for cellular, the acp_RoutingTable.hpp can be included.
17 * The following functionalities are provided by the API.
18 *
19 * # Description of ReadSimCardInfo function
20 * This function allows the user to get sim card related information, e.g., iccid, imisi, mdn, etc.
21 * The SimCardInfo constructor defines default values for the members
22 * User must provide the reference to a SimCardInfo_t object
23 *
24 * # Description of ReadModemInfo function
25 * This function allows the user to get modem related information, e.g., manufacturer, imei, firmware, etc.
26 * The ModemInfo constructor defines default values for the members.
27 * User must provide the reference to a ModemInfo_t object
28 *
29 * # Description of ReadRadioStatus function
30 * This function allows the user to get the current Radio status information, e.g., RAT, registration state, signal strength, etc.
31 * The RadioStatus constructor defines default values for the members.
32 * User must provide the reference to a RadioStatus_t object
33 *
34 * # Description of CreateConnectionProfile function
35 * This function allows the user to create a profile on the modem.
36 * The DeviceProfile constructor defines default values for the members.
37 * User must provide the reference to a DeviceProfile_t object
38 *
39 * # Description of DeleteConnectionProfile function
40 * This function allows the user to delete a profile from the modem.
41 * User must provide the profileid to be deleted
42 *
43 * # Description of ModifyConnectionProfile function
44 * This function allows the user to modify an existing profile on the modem.
45 * The DeviceProfile constructor defines default values for the members.
46 * User must provide the reference to a DeviceProfile_t object
47 *
48 * # Description of DisplayAllConnectionProfile function
49 * This function allows the user to view all the profiles present on the device.
50 *
51 * # Description of DisplayParticularConnectionProfile function
52 * This function allows the user to view a specific profile on the device.
53 * The user must specify the profileid they want to view.
54 * The profileid must be within the supported range of the device.
55 *
56 * # Description of GetConnectionStatus function
57 * This function allows the user to get the cellular connection related information, e.g., interface, ipv4/6 addreses, ipv4/6 gateways, etc.
58 * The ConnectionStatus constructor defines default values for the members
59 * User must provide the reference to a ConnectionStatus_t object
60 *
61 * # Description of RadioOn function
62 * This function allows the user to turn the radio service on.
63 *
64 * # Description of RadioOff function
65 * This function allows the user to turn the radio service off.
66 * User might get ACPEDGE_RET_FAILURE error code while trying to access radio.
67 *
68 * # Description of AttachDeviceGPRS function
69 * This function allows the user to attach the device to the packet domain service.
70 *
71 * # Description of DetachDeviceGPRS function
72 * This function allows the user to detach the device to the packet domain servicem.
73 *
74 * # Description of ResetModem function
75 * This function allows the user to reset the modem.
76 *
77 * # Description of SetExtendedPassGNSS function
78 * This function allows the user to set the extended AT password to access GNSS commands.
79 *
80 * # Description of InputExtendedPassGNSS function
81 * This function allows the user to enter the extended AT password to execute GNSS commands.
82 *
83 * # Description of DisableGNSSDynamicPowerOptimization function
84 * This function allows the user to disable the Dynamic Power Optimization mode.
85 *
86 * # Description of DisableGNSSLNAPin function
87 * This function allows the user to disable LNA in Qualcomm’s GNSS engine.
88 *
89 * # Description of DisableGNSSAutoStart function
90 * This function allows the user to disable autostart in Qualcomm’s GNSS engine.
91 * Any setting change takes effect next time the modem is reset.
92 *
93 * # Description of ConnectDataSession function
94 * This function allows the user to establish a cellular data session.
95 * The user must provide the profileid and the data session technology.
96 * The intended profile can be created or modified on the device before calling this function.
97 *
98 * # Description of DisconnectDataSession function
99 * This function allows the user to disconnect current data session.
100 * User must provide the profileid used for connecting the data session.
101 *
102 * # Description of IssueATCommands function
103 * This function allows the user to issue a single or multiple at commands to the device.
104 * User must provide the list of AT command as strings and a reference to another list of strings to get the output.
105 *
106 * # Description of GetDataBearerTechnology function
107 * This function allows user to get the data bearer technology.
108 *
109 * # Description of Set and Get cell clock time functions
110 * These functions allows the user to get or set the cell time.
111 *
112 * # Description of Set eDRX state
113 * This function allows the user to enable or disable eDRX stateg.
114 *
115 *
116 * The following use cases are supported by the API.
117 *
118 * # Read information from the Sim card
119 *
120 * # Read information from the modem
121 *
122 * # Read current operational(Radio) status of the modem
123 *
124 * # Create, modify and delete profiles on the device
125 *
126 * # Display all or particular profile present on the device
127 *
128 * # Read connection status from the system.
129 * Can be used to verify a connection
130 *
131 * # Turn ON or OFF the radio service
132 *
133 * # Reset Modem
134 * Some setting changes require the modem reset to take effect, e.g., GNSS autostart.
135 *
136 * # Remove or unlock Sim PIN
137 *
138 * # Disable GNSS Dynamic Power Optimization (DPO)
139 * When DPO is enabled and the GNSS signal is strong, the GNSS RF and clocks will be
140 * turned off in-between fixes during a tracking session. For example, with a 1 second
141 * fix rate, if a GNSS fix is obtained within 200 ms then the GNSS RF and clocks will be
142 * shut down for the next 800 ms. This behavior is intended to save current when running
143 * from a battery – such as a cell phone.
144 *
145 * # Disable GNSS Autostart and LNA pin
146 *
147 * # Connect or Disconnect cellular data session
148 *
149 * # Issue single or multiple AT commands to the modem
150 * # Set and get cellular time
151 * # Enable or disable eDRX state
152 *
153 *
154 *
155 * Sequence of API calls:
156 *
157 * RadioOn() - full functionality, attaches the device to PDS
158 * CreateConnectionProfile() - Allows user to create their own profile
159 * ConnectDataSession() - Checks if the radio power is ON, profile status is active, gets connection status, then proceeds for data connection
160 * DisconnectDataSession - Disconnects from data session, detaches the device from PDS, then turns the radio power OFF.
161 *
162 */
