12  * This header is the interface of I2C API.
13  * The user will be able to write to and read from slave devices over I2C via the provided interfaces.
14  * I2C stands for Inter-Integrated Circuit. It is a synchronous, multi-master, multi-slave, serial bus.
15  * This protocol is intended to allow multiple slave digital integrated circuits (chips) to communicate
16  * with one or more master chips.
17  *
18  * The user does not need to add any other header to use this API.
19  *
20  * The following functionalities are inherited from SerialBus class.
21  *
22  * # Description of GetFileDescriptor function
23  * This function allows the user to get the reference to an open device
24  *
25  * # Description of IsValid function
26  * This function allows the user to check if the file descriptor is valid
27  *
28  * # Description of GetFileName function
29  * This function allows the user to get the name of the resource to open (i.e. /dev/ttyAT)
30  *
31  * The following functionalities are provided by the API.
32  *
33  * # Description of SetAddress function
34  * This function allows the user to set the I2C address of the slave device
35  * The user must specify the 7-bit I2C address of the slave device.
36  *
37  * # Description of WriteSerial function (1)
38  * This function allows the user to write a string to a slave device over the I2C bus
39  * The user must specify the string to be written.
40  *
41  * # Description of WriteSerial function (2)
42  * This function allows the user to write a C array to a slave device over the I2C bus
43  * The user must specify the data array to be written.
44  * The user must specify the number of bytes to be written.
45  *
46  * # Description of WriteRegister function
47  * This function allows the user to write to a register of a slave device over the I2C bus
48  * The user must specify the address of a register within the slave device.
49  * The user must specify the single-byte value to be written.
50  *
51  * # Description of ReadSerial function
52  * This function allows the user to read from a device connected to the I2C bus
53  * The user must specify the array where input will be copied.
54  * The user must specify the number of bytes to be copied.
55  *
56  * # Description of ReadRegister function
57  * This function allows the user to read from a register of a slave device over the I2C bus
58  * The user must specify the address of a register within the slave device.
59  * The user must specify the single-byte variable where input will be copied.
60  *
61  * The following use cases are supported by the API.
62  *
63  * # Configure a slave device over I2C
64  * The user configures a slave device by writing to the device setup registers.
65  *
66  * # Reading a mesurement from a sensor over I2C
67  * The user reads a mesurement from a sensor over I2C.
68  *
69  * # Writing a command to a driver over I2C
70  * The user writes a command to a driver over I2C.
71  *
72  */
