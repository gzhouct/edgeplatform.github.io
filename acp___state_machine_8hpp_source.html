12  * This header is the interface for using a State Machine
13  * The user will be to transition from one state to another state
14  *
15  * The user does not need to add any other header to use this API.
16  *
17  * The following functionalities are available from the AcpStateMachine class.
18  *
19  * # Description of AcpStateMachine method
20  * This function constructs a AcpStateMachine object.
21  *
22  * # Description of AddState method
23  * This function adds a new state with or without a broadcast event
24  *
25  * # Description of RemoveState method
26  * This function removes a state
27  *
28  * # Description of AddBroadcastToState method
29  * This function adds a broadcast event to an existing state. If the state doesn't exist, it will be created
30  *
31  * # Description of RemoveBroadcastFromState method
32  * This function removes a broadcast event from an existing state.
33  *
34  * # Description of GetCurrentState method
35  * This function get the name of the current state.
36  *
37  * # Description of AddTransition method
38  * This function adds a transition from one state to another.
39  *
40  * # Description of GetMachineName method
41  * This function gets the name of the state machine.
42  *
43  * # Description of StateTransitionCallback method
44  * This function creates a state transition callback.
45  *
46  * # Description of TransitionToState method
47  * This function transits to the next state. Events will be broadcasted after transition.
48  *
49  */
