This method configures and enables an ADC as a boot source It is possible to specify a single range or two ranges of operation with a non-operational range in between:

Condition: Range: (above_adc_reading_in < below_adc_reading_in) adc_reading is in [above_adc_reading_in, below_adc_reading_in] (above_adc_reading_in > below_adc_reading_in) adc_reading is less than (<) below_adc_reading_in, and adc_reading is greater than (>) above_adc_reading_in

Parameters
[in]adc_number_inThe number of the ADC to configure
[in]polling_interval_ms_inHow frequently to poll the ADC while sleeping
[in]above_adc_reading_inReading above which the system should boot
[in]below_adc_reading_inReading below which the system should boot
Returns
true The ADC is configured and enabled as boot source
Note
This interface is not thread-safe

◆ GetWakeUpReason()

AcpWakeUpReason_t aep::api::PowerManager::GetWakeUpReason ( ) const

This method gets the reason for the last OS boot (wake up) Boot may occur due to exit from ultra low power mode or press on Power button Exit from low power state may occur through various resources

The wake up reasons are queried in the following order:

  1. Timer expired
  2. GPIO pin 36 was toggled
  3. GPIO pin 38 was toggled
  4. ADC2 reading was in range
  5. ADC3 reading was in range

The wake-up reason is set to power on if none of the above conditions were true

Returns
The wake up reason for the last OS boot
Note
This interface is not thread-safe
