12  * This header is the interface of power strategy API.
13  * The user will be able to configure hardware sources to boot the OS from ultra low power mode (ULPM)
14  * The user will be able to track the reasons to enter in ULPM. This action is also known as sleep.
15  * The user will be able to track the reasons to exit from ULPM. This action is also known as wake up.
16  * The user shall use this interface to interact with the following services:
17  *
18  * 1) GPIO API
19  * 2) ADC Channel API
20  * 3) Ultra Low Power Mode (ULPM) API
21  * 4) Boot Reason Query API
22  *
23  * The user does not need to add any other header to use this API.
24  *
25  * The following functionalities are provided by the API.
26  *
27  * # Description of Set power supply mode function
28  * This function allows the user to set the power supply mode
29  * The user must specify the power mode
30  *
31  * # Description of Get power supply voltage function
32  * This function allows the user to get the power supply voltage in volts
33  *
34  * # Description of Set sleep reason function
35  * This function allows the user to set the reason why the device goes to sleep into non-volatile memory (NVM)
36  * The user must specify the sleep reason
37  *
38  * # Description of Get sleep reason function
39  * This function allows the user to get the sleep reason from non-volatile memory (NVM)
40  *
41  * # Description of Remove sleep reason function
42  * This function allows the user to remove the sleep reason from non-volatile memory (NVM)
43  *
44  * # Description of Get wake up reason function
45  * This function allows the user to get the reason for the last OS boot (wake up)
46  *
47  * # Description of Boot on timer function
48  * This function allows the user to set the expiration value of the boot timer
49  * The user must specify the expiration time (in seconds) to boot
50  *
51  * # Description of Boot on GPIO function
52  * This function allows the user to set the state of the GPIO pin to boot
53  * The user must specify the GPIO number to boot
54  * The user must specify the GPIO state which should cause boot
55  *
56  * # Description of Boot on ADC function
57  * This function allows the user to configure and to enable an ADC as a boot source
58  * The user must specify the number of the ADC to configure
59  * The user must specify how frequently to poll the ADC while sleeping
60  * The user must specify the reading above which the system should boot
61  * The user must specify the reading below which the system should boot
62  *
63  * The following use cases are supported by the API.
64  *
65  * # Configure the power supply mode
66  * The user configures the power supply mode
67  *
68  * # Read the power supply voltage
69  * The user reads the power supply voltage in volts
70  *
71  * # Configure hardware sources to boot the OS from ultra low power mode (ULPM)
72  * The user configures a Timer, a GPIO pin and/or a ADC channel to boot the OS from ultra low power mode (ULPM)
73  *
74  * # Track the reasons to enter in ULPM (to go to sleep)
75  * The user writes, reads and deletes the sleep reason from non-volatile memory (NVM)
76  *
77  * # Track the reasons to exit from ULPM (to wake up/to boot the OS)
78  * The user reads the reason for the last OS boot (wake up)
79  *
80  */
